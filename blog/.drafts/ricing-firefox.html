Firefox is, in my opinion, the most extensively customizable modern mainstream web browser. The customizability of Firefox is, in fact, the sole reason for which I prefer Firefox to, say, Ungoogled Chromium. There are a few good guides out there, which I will be linking at the beginning of each section, but they are scattered about. I think there should be one universal and encompassing guide to ricing Firefox.

I) Homepage/New Tab Page

Relevant Links: /wg/ thread, a pastebin, Startpage Emporium
To start off, we will be reviewing one of the two of these four sections which is not specific to Firefox: homepages. The concept is very simple: a webpage which is functionally useful when you open a blank Firefox window. It may include anything. Common elements include:
-Links to frequently visited sites
-Search bar
-Weather
-Autistic /biz/ crypto prices
-Waifus telling you to have a nice day
Point is, there is plenty of creativity you can use here, and its not rare that I will see someone do something new and cool in a homepage. To create a basic, /wg/ ready homepage, you can simply make one using HTML+CSS (although you may need to write JS scripts for some things, like weather). This can be done fairly easily. If you're lazy, you can perform the age-old art of stealing other people's shit. See above, there are tons and tons of homepages people make. I'm currently using one I stole off of someone else (and changed a bit) but I am appalled at how bloated it is and I will probably write my own soon. Once you have a homepage you like, it's time to apply it. You can host the html file wherever you want. I recommend hosting it on Github Pages . Once it's hosted, go to about:preferences, and change Homepage and new windows to Custom URLs.... This is all you need to do if you just want to change your homepage. However, chances are, you also want to change your newtab. Unfortunately, Firefox removed our ability to do custom newtabs a few updates ago, so I use the plugin Custom New Tab to change the newtab page. Just set the newtab to your hosted file, and you should be good to go.

II) Stylus

Relevant Links: Userstyles
In case you're unaware, Stylus is a plugin for Firefox and Chrome that lets you use your own custom CSS on websites. The tool is pretty cool and can lead to some very nice site designs. Stylus also lets you use Global Styles, which is css that will apply to every website you visit. While it may sound good to have a consistent style among all websites you visit, you soon realize that Global Styles are autistic and will make 80% of websites look like garbage (seriously, just don't use them). Admittedly, Stylus can be somewhat difficult to use sometimes. It requires you to study the website's CSS and mostly involves a lot of tinkering. I recommend writing CSS for only websites you frequent, if you value your time. Of course, userstyles.org has a ton of good styles you can steal. III) userchrome.css

Relevant Links: Userchrome.org, /r/Firefoxcss Userchrome.css is one one of the best things about ricing Firefox. It lets you change the way the browser's interface looks, including (almost) everything from tabs to scrollbars. It can be a little tricky to actually get started with a userChrome.css file. Create userchrome.css in ~/.mozilla/chrome (create this folder if necessary.). To be honest, userchrome.css can be much more difficult to tinker with than site styles. You will have to become familiar with the wack-ass Firefox CSS elements and, if you want to do anything other than recolor stuff on the browser, you will require a more extended knowledge of CSS. That is why I recommend stealing something off of /r/Firefoxcss and modifying it to your liking (Sorting by top...all time can give you actual themes and not just questions). Me, being the lazy fuaarrrrk that I am use a modified version of this.

Themes

Last, but not least (actually, yes, least) are Firefox themes. Firefox themes are necessary to color things like the URL bar which cannot be changed by userchrome.css. You can find many Firefox themes on the Firefox Addons Catalog However, you will find that LITERALLY EVERY SINGLE ONE OF THESE is garbage. Seriously, it's astounding how bad these are. So, what we will do is make our own Firefox theme. Making a theme isn't too bad, but applying one can be a pain. To make a theme, you should go about like you were making any Firefox addon. Create a manifest.json file and populate it like this: (this is my theme) 


{
"manifest_version": 2,
"name": "fft",
"version": "0.3",
"theme": {
    "colors": {
       "accentcolor": "#0c0c0c",
       "textcolor": "white",
       "toolbar": "#0c0c0c",
       "toolbar_text": "white",
       "toolbar_field": "#0c0c0c",
       "toolbar_field_border": "#0c0c0c",
       "toolbar_field_text": "white",
       "toolbar_top_separator": "#ffa64f",
       "toolbar_bottom_separator": "#0c0c0c",
       "toolbar_vertical_separator": "#0c0c0c"
    }
  }
}
Then package it into a .zip or .xpi:
zip -r -FS theme.zip *
Now that you have this, we can "test" out the theme.
  • 1) Go to about:debugging
  • 2) Click "Load Temporary Add-On"
  • 3) Choose your file.
HOWEVER!!!
Doing this, every time you restart Firefox, you will have lost your theme. So in order to avert this ill fate, we will (prepare yourself) PUBLISH OUR THEME. Thanks """MOZ://A""". So, basically you will have to go through a small process to get your theme published.
  • 1)Visit Firefox Addons and make a """MOZ://A""" account
  • 2)Go to the developer club and click Submit a New Add-on
